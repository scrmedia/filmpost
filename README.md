# FilmPost

FilmPost is a web application built for wedding videographers that automates the creation and publishing of SEO-optimised content. After filming a wedding, a videographer uploads their video to FilmPost, answers a short questionnaire about the venue, and the app generates a YouTube video title and description, a full blog post, and publishes both automatically — without any manual writing or copying.

---

## What it does

### Upload flow (4-step wizard)

**Step 1 — Select video**
The user drags and drops (or clicks to browse) a video file. There is no size limit imposed by the app; large files are handled via a chunked upload proxy.

**Step 2 — Venue details**
The user fills in a short questionnaire: venue name, location, wedding season, overall mood, key moments, and any other details. A hero image can also be uploaded here; it becomes the WordPress featured image and the YouTube thumbnail.

**Step 3 — Review and edit**
FilmPost sends the questionnaire answers to Claude (Anthropic) and receives back:
- A YouTube-optimised video title
- A YouTube description with timestamps, hashtags, and location keywords
- A full HTML blog post (600–900 words) written in the user's saved brand voice

All three are fully editable before publishing. The blog post includes an inline AI rewriter: the user can highlight any passage, type a rewrite instruction (e.g. "make this more formal" or "add more detail about the ceremony"), and accept or reject the revision. Rejected revisions return to the previous version. Every accepted rewrite is stored in a version history in Supabase so the user can compare changes over time.

**Step 4 — Publish**
On confirmation, FilmPost:
1. Creates a WordPress draft via the WordPress REST API, sets the featured image, and returns an edit link
2. Initiates a YouTube resumable upload session on the server
3. Streams the video to YouTube in 4 MB chunks through a Vercel serverless proxy (to avoid CORS restrictions)
4. Once the YouTube upload completes, fetches the video ID and injects a responsive iframe embed into the WordPress draft
5. Updates the Supabase post record with the YouTube URL and WordPress edit URL

Progress is shown in real time with a percentage bar.

---

### AI content generation

Content is generated by Claude claude-sonnet-4-6 via a thin Vercel serverless proxy (`/api/claude`). The prompt is constructed from the venue questionnaire answers plus several user-level signals:

- **Tone of voice** — a free-text field on the Profile page. When populated, it is injected into the system prompt as a brand voice instruction ("Write in a style that reflects this brand voice: …"), influencing vocabulary, sentence structure, and personality across both the blog post and the YouTube description.
- **SEO plugin** — if the user has Yoast, Rank Math, or All in One SEO configured, the blog post output includes a plugin-specific metadata block at the end with the exact fields those plugins expect (SEO title, meta description, focus keyphrase, slug). This block is clearly labelled so the user can copy and paste it directly into their plugin.
- **YouTube embed** — after the video is uploaded and a video ID is available, an iframe embed is inserted after the opening paragraph of the blog post before the WordPress draft is updated.

The prompt also includes the user's business name and a GEO/SEO optimisation layer: venue name and location are woven into headings, the opening paragraph, and alt-text suggestions.

---

### Blog post rewriter

Inside the Review step, the user can select any portion of the generated blog post and request a targeted rewrite. The rewriter:

- Sends only the selected passage to Claude with the user's instruction
- Returns the revised passage with word-level diff highlighting (additions in green, deletions in red strikethrough)
- Shows Accept and Reject buttons side by side
- On Accept: replaces the passage in the editor and appends a version snapshot to `blog_versions` in Supabase
- On Reject: discards the revision and restores the original

Version history is accessible in the History page for each post.

---

### History

The History page lists all posts saved to the user's account, retrieved from the `posts` Supabase table. Each row shows the venue name, date, status, and links to the WordPress draft and YouTube video. Expanding a post shows all saved blog versions with timestamps and the ability to compare them with word-level diffs.

---

### Dashboard

The Dashboard shows four stat cards:

- **Videos Published** — count of posts in the user's history
- **Blog Posts Created** — count of posts with a saved WordPress edit URL
- **This Month** — posts created in the current calendar month
- **Avg per Month** — average posts per month across the account lifetime

Below the stats is a Recent Uploads list (last 5 posts) and a Real Activity feed derived from actual post data, showing YouTube uploads and WordPress draft creations with relative timestamps. A quick-action panel at the bottom shows the connection status of the YouTube account and shortcuts to New Upload.

---

### Settings

The Settings page manages integrations:

**YouTube**
Clicking Connect initiates a Google OAuth 2.0 flow (authorization code grant with offline access). The user is redirected to Google's consent screen, then back to `/youtube/callback`. The callback page exchanges the code for an access token and refresh token via `/api/youtube-auth`, stores both in Supabase, and displays the connected channel name. Disconnect clears the tokens from Supabase.

**WordPress**
An inline form accepts the site URL, WordPress username, and an Application Password. On Save, FilmPost verifies the credentials against the WordPress REST API before storing them. The connected URL is shown persistently. Disconnect clears all three fields from Supabase.

### Profile

The Profile page stores business information used in content generation:

| Field | Purpose |
|---|---|
| Business name | Injected into blog post byline and YouTube description |
| Tagline | Optional brand line used in post footers |
| Enquiry email | Added to blog post calls to action |
| Website | Linked in YouTube description and blog post footer |
| Instagram, TikTok, Facebook | Social links added to the YouTube description |
| Tone of voice | Free-text brand voice used in Claude prompts |
| SEO plugin | Drives which metadata block is appended to blog posts |
| Platform | Future use for platform-specific publish flows |

## Authentication

FilmPost uses a custom email/password authentication system built on top of Supabase. Passwords are hashed with bcryptjs (cost factor 10) and stored in a `password_hash` column. There is no Supabase Auth — all auth is handled by the app itself using direct table queries and localStorage session persistence.

## Architecture

| Layer | Technology |
|---|---|
| Frontend | React 18 SPA (Create React App) |
| Hosting | Vercel (static build + serverless functions) |
| Serverless API | Vercel Functions (`/api/*.js`) |
| Database | Supabase (Postgres) |
| AI | Anthropic Claude API (`claude-sonnet-4-6`) |
| YouTube publishing | YouTube Data API v3 (resumable upload) |
| WordPress publishing | WordPress REST API v2 (Application Passwords) |
| Password hashing | bcryptjs |

---

## Serverless API endpoints

| Endpoint | Purpose |
|---|---|
| `POST /api/claude` | Proxy to Anthropic Messages API |
| `POST /api/youtube-auth` | YouTube OAuth: get auth URL, exchange code, refresh token |
| `POST /api/youtube-upload` | Initiate a YouTube resumable upload session |
| `POST /api/youtube-upload-chunk` | Proxy a single 4 MB video chunk to YouTube (avoids CORS) |
| `POST /api/youtube-thumbnail` | Upload a thumbnail image to YouTube via the Data API |

## Database schema

### `users`
Stores account credentials, business profile, integration tokens, and publishing preferences.

Key columns: `email`, `password_hash`, `business_name`, `tagline`, `enquiry_email`, `website`, `instagram`, `tiktok`, `facebook`, `tone_of_voice`, `seo_plugin`, `platform`, `wp_url`, `wp_user`, `wp_pass`, `youtube_access_token`, `youtube_refresh_token`, `youtube_channel_name`, `featured_opt_in`, `created_at`

### `posts`
One row per upload. Stores all generated content and publish results.

Key columns: `user_id`, `venue_name`, `youtube_title`, `youtube_description`, `blog_content`, `blog_versions` (jsonb array of version snapshots), `wp_edit_url`, `youtube_url`, `target_keyword`, `location`, `status`, `created_at`

### `venues` *(schema ready, UI coming soon)*
A reusable venue library. Saved venues can pre-fill the questionnaire on future uploads.

Columns: `id`, `user_id`, `venue_name`, `location`, `style_notes`, `lighting_notes`, `prefill_data` (jsonb), `created_at`

## Required environment variables

Set these in Vercel (or `.env.local` for local development):

```
ANTHROPIC_API_KEY=
YOUTUBE_CLIENT_ID=
YOUTUBE_CLIENT_SECRET=
YOUTUBE_REDIRECT_URI=https://filmpost.vercel.app/youtube/callback
REACT_APP_SUPABASE_URL=
REACT_APP_SUPABASE_ANON_KEY=
```

## Local development

```bash
npm install
npm start        # React dev server on http://localhost:3000
vercel dev       # Runs serverless functions alongside the React app
```

The YouTube OAuth redirect URI must match exactly. For local development, add `http://localhost:3000/youtube/callback` as an authorised redirect URI in Google Cloud Console and set `YOUTUBE_REDIRECT_URI` accordingly.
